## Scalability Considerations

### Current (Single Node)
- Supports 1 ESP32 sensor node
- Single FastAPI instance
- Single MongoDB instance
- Local deployment

### Future (Multi-Node)
- Load balancer (Nginx/Traefik)
- Multiple FastAPI instances
- MongoDB replica set
- Redis for caching
- Message queue (RabbitMQ/Kafka)
- Multiple sensor nodes
- Microservices architecture

## Deployment Options

### Development (Current)
- Local machine
- MongoDB local/Docker
- npm run dev
- python main.py

### Production
- Cloud VM (AWS EC2, DigitalOcean, etc.)
- MongoDB Atlas
- Docker containers
- Process manager (PM2/Systemd)
- Reverse proxy (Nginx)
- SSL/TLS certificates
- Domain name

### Docker Deployment
```yaml
services:
  mongodb:
    image: mongo:latest
  
  backend:
    build: ./backend
    depends_on:
      - mongodb
  
  frontend:
    build: ./frontend
    depends_on:
      - backend
  
  nginx:
    image: nginx:latest
    depends_on:
      - frontend
```

## Performance Characteristics

- **API Response Time**: <100ms (typical)
- **Database Query Time**: <50ms (typical)
- **AI Analysis Time**: 1-3s (OpenAI), <100ms (rule-based)
- **WebSocket Latency**: <50ms
- **Sensor Reading Interval**: 2s (configurable)
- **Dashboard Refresh**: 10s (configurable)
- **Alert Generation**: Real-time
- **Concurrent Users**: 100+ (single instance)

## Data Retention

- **Sensor Data**: 90 days (configurable)
- **Alerts**: 1 year
- **Risk Analysis**: 30 days
- **Sprinkler Logs**: 1 year
- **User Sessions**: 30 minutes (JWT expiry)
